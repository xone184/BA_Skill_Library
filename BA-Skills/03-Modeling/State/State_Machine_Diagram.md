# Skill Name
State Machine Diagram

## Level
Level 3 - Architecture Skill

## Purpose
Thiết kế vòng đời (Lifecycle) của một đối tượng dữ liệu.

## Inputs
- Thực thể dữ liệu (vd: Hóa đơn, Đơn hàng)

## Process
1. Xác định các trạng thái (States)
2. Xác định sự kiện kích hoạt (Triggers/Events)
3. Định nghĩa trạng thái Initial và Final
4. Vẽ mũi tên chuyển đổi (Transitions)

## Outputs
- State Diagram (Mermaid)

## Checklist
- [ ] Có trạng thái Cancelled/Failed không?
- [ ] Có luồng quay lui (Rollback) trạng thái không?
